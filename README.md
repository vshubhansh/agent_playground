# agent_playground

A playground for agent-driven experiments — a space deliberately kept open for an AI coding agent to design, build, and refine ideas with a large degree of creative freedom.

This repository is **the first of many such experiments** to live here. Each one starts from a loose, open-ended brief rather than a rigid spec, and the goal is as much to see *what an agent chooses to build* as it is to ship a finished artifact. Expect the contents to grow over time as new experiments are added alongside the existing ones.

## Experiments

### 1. The Craft Collection — `skills/`

A self-authored set of five portable [Cursor Agent Skills](https://cursor.com), built from a blank-canvas brief: *"this is your playground… create any skill that you would want… show me your creative side."*

They are unified by one idea — **act in code with respect for time and consequence** — and each targets a distinct relationship a change has with time:

| Skill | Relationship | In one line |
|-------|--------------|-------------|
| `git-archaeologist` | the past | Recover lost intent from git history |
| `chestertons-fence` | the present | Don't remove code whose purpose you don't understand |
| `assumption-ledger` | the self | Know what you actually know vs. what you assume |
| `measure-twice` | the action | Weigh blast radius x reversibility before cutting (keystone) |
| `leave-a-trail` | the future | Preserve intent forward via commits, PRs, and ADRs |

See [skills/README.md](skills/README.md) for the full story, design notes, usage instructions, and worked examples.

## Conventions

- Experiments are self-contained in their own top-level directory, each with its own README.
- Work favors portability and clarity, so anything built here can be read, shared, or reused on its own.
